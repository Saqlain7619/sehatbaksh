# MedPlus Medical Website

A modern, multilingual, and responsive medical website for healthcare providers, designed to deliver an excellent user experience for patients and staff alike.

---

## 🚀 Features

- **Responsive Design:** Works seamlessly on desktop and mobile.
- **Navigation with Icons:** FontAwesome icons for all main navigation links (desktop and mobile).
- **Profile Management:** Avatar/profile picture sync across all pages using localStorage.
- **Appointment Booking:** Book appointments with doctors in specific departments.
- **Health Library & Disease Search:** Educational resources and search tools for users.
- **Multilingual Support:** Google Translate integration with custom selector for English and Urdu; RTL support for Urdu.
- **Modern UI/UX:** Glassy buttons, gradients, and animated transitions (Animate.css).
- **Accessibility:** ARIA labels, semantic HTML, and notranslate attributes for branding.
- **Privacy & Security:** Privacy Policy, Terms of Service, and secure profile/logout management.

## 📁 Project Structure

- `index.html` - Home page
- `doctors.html` - Doctors and appointment booking
- `departments.html` - Departments listing
- `blog.html` - Blog articles
- `contact.html` - Contact form
- `health-library/index.html` - Health library
- `diseases/disease-search.html` - Disease search tool
- `settings.html`, `profile.html`, etc. - User management
- `styles/` - CSS files
- `scripts/` - JavaScript files
- `images/` - Images and avatars

## 🚀 Deployment Instructions

### Local Deployment
1. **Clone or Download** the project folder to your computer.
2. **Open** `index.html` (or any page) directly in your web browser.
   - No server or build process is required—everything runs client-side.

### Deploying Online (Netlify, Vercel, GitHub Pages, etc.)
1. **Sign up** for a static hosting provider (e.g., [Netlify](https://netlify.com), [Vercel](https://vercel.com), or [GitHub Pages](https://pages.github.com)).
2. **Upload** your project folder or connect your Git repository.
3. **Configure** the publish directory to the root folder (where `index.html` is located).
4. **Deploy** the site and share your public URL!
